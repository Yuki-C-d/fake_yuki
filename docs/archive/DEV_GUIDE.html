📝 每个文件说明

app/__init__.py

  • 空文件，不用写代码
  • 作用：让 Python 把 app 当成"包"，其他文件才能 from app.xxx import yyy
  • ❌ 别删！

apps/music/backend/config.py — 路径配置

  • 存放项目里所有路径常量
  • 以后引路径都从这里拿，改一处全局生效
import os

# 项目根目录（D:\music-vault）
BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))

# 音乐文件夹
MUSIC_DIR = os.path.join(BASE_DIR, "music")

# 数据库文件路径
DB_PATH = os.path.join(BASE_DIR, "data", "music.db")

apps/music/backend/models.py — 数据库 ✅

  • 3 张表：songs（歌曲）、playlists（歌单）、playlist_songs（歌单-歌曲关联）
  • 外键级联：删除歌单自动清理关联，删除歌曲自动清理关联
  • PRAGMA foreign_keys = ON 确保外键生效（SQLite 默认关闭）
  • 使用 Python 内置 sqlite3，INSERT OR IGNORE 防重复

apps/music/backend/scanner.py — 扫描器 ✅

  • 遍历 music 文件夹 → 自动检测 FLAC/MP4 → 用 mutagen 读标签 → 写入数据库
  • get_tag() 跨格式读标签（FLAC 用 title/artist，MP4 用 ©nam/©ART）
  • 运行：python -c "from apps.music.backend.scanner import scan; scan()"

apps/music/backend/main.py ⭐ API 服务器入口

  • FastAPI 启动入口，含 16 个路由
  • 启动：cd D:\music-vault && python -m uvicorn apps.music.backend.main:app --host 0.0.0.0 --port 8080
  • 启动后自动 init_db() + scan()，无需手动跑 python 命令

  • Phase 3 新增（2026-07-07）：

🔧 ncmdump 解密工具

位置

D:\music-vault\ncmdump\ncmdump.exe

解密单个文件

cd D:\music-vault
.\ncmdump\ncmdump.exe "music\歌曲名.ncm" -o music

批量解密整个文件夹

cd D:\music-vault
.\ncmdump\ncmdump.exe -d music -o music
💡 现在不用手动运行 ncmdump 了！直接在网页上拖 .ncm 文件，后端自动解密+入库。

📤 网页上传歌曲（推荐）

💡 最简单的方式：打开网页 http://127.0.0.1:8080，拖拽音频文件到虚线框即可。

支持的上传格式

格式处理方式结果
.m4a .mp3 .ogg .wav .aac直接入库✅ 立即可播
.ncmncmdump 自动解密 → 检测格式✅ 或 🔧（看下方）
.flac（真）直接入库（头4字节=fLaC）✅ 可播
.flac（AV3A伪装）自动后台转码（头4字节≠fLaC）🔧 30-50分钟后可播

上传流程

拖文件到网页 → 上传 → 解密(如.ncm) → 格式检测
  ├─ 真音频 → 直接入库 → 歌单出现 → ✅
  └─ AV3A   → 后台自动转码 → 转码队列显示进度
               → 完成后自动入库 → 歌单出现 → ✅

转码进度

网页下方有「🔄 转码中」区域，每 30 秒自动刷新：

状态含义
⏳ 排队中等待处理（同时最多 N 首并行）
🔓 解码中…av3a_decoder 解码中（最慢，30-50分钟）
🎚️ 即将完成ffmpeg 编码中（~30秒）
✅ 已完成自动入库，刷新后出现在歌单

🎛️ AV3A/MP4 → AAC/M4A 转码

⚠️ 某些音乐平台下载的文件是 MP4 容器 + AV3A (Audio Vivid/菁彩声) 编码，浏览器无法解码。需转码为 AAC/M4A 才能播放。

前置条件

# 确认有 av3a_decoder.exe（下载地址见下）
ls tools/av3a_decoder/av3a_decoder.exe

# 确认有 ffmpeg
ffmpeg -version

一键转码（支持并行）

cd D:\music-vault
bash convert_to_m4a.sh
# 默认并行数=CPU核心数，可覆盖：
# MAX_JOBS=2 bash convert_to_m4a.sh
💡 现在上传 AV3A 后会自动后台转码，通常不需要手动运行此脚本。脚本依然可用，适合批量处理。

流水线（每首歌自动完成）

步骤工具说明耗时
1. 提取Python (内联)MP4 mdat box → raw AV3A 流秒级
2. 解码av3a_decoder.exeAV3A 12ch → WAV30-50 分钟/首
3. 编码ffmpegWAV → AAC 256k 立体声 M4A + 元数据5-10 秒

⚠️ 常见问题

Q: 浏览器点播放没声音

原因：文件可能是 AV3A（菁彩声/AVS3-P3）编码，不是标准音频格式。浏览器不支持 AV3A 解码。

✅ 已解决：使用 convert_to_m4a.sh 一键转码为 AAC/M4A 即可。见上方「🎛️ AV3A 转码」章节。

Q: 新歌不显示在网页上

解决：把文件放到 music/ 目录下，然后触发扫描：

curl -X POST http://127.0.0.1:8080/api/scan

或者重启服务器（启动时自动扫描）。

Q: 播放器显示"格式不支持"

原因1：VS Code 内置浏览器不支持 M4A 解码。用外部浏览器（Chrome/Edge）打开。

原因2：文件是 AV3A 编码。系统会自动检测→后台转码→完成后可播。查看页面下方转码进度。

Q: scanner 报 "not a valid FLAC file"

原因：文件扩展名是 .flac 但实际是 MP4/AV3A 容器。scanner 已加入 AV3A 检测，伪 FLAC 自动跳过不入库。
